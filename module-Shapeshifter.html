(static) branch_child_collect(node_list, selected_node) → {array}

Returns the non-leaf children below a particular node. Unlike child_collect, this excludes leaf children. If the selected node happens to be a leaf node, this will return an empty array. Does this by recursively, using pass by sharing (like reference) to return values to caller Note that in order to prevent double counting, it must maintain two lists - one to loop, and another to accumulate. Otherwise, it would be adding visited nodes to the list of unvisited nodes.

(static) create_flat_tree(flat) → {object}

Creates a object representing a flat arrangement of objects. The hierarchical structure is recorded in the item called 'children', which is an array containing only the ids of the immediate children. These do not in turn contain their own children. The overall structure is flat to make it easy to do a linear search rather than having to do a proper depth- or breadth-first search. The first element is assumed to be the root. A future feature will be to incorporate some options to add some flexibility to the structures read in and produced. The input is a flat-file, probably originally from a csv (parsed separately into an array of objects where each element is named according to the csv header)
